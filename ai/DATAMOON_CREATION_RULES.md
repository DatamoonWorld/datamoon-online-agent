# DATAMOON_CREATION_RULES.md

# Objetivo

Este documento define as regras oficiais para criação de novas espécies de Datamoons.

Seu objetivo é garantir consistência entre:

- Lore;
- Gameplay;
- Combate;
- Progressão;
- Ecossistemas;
- Arte;
- Narrativa.

Todo Datamoon criado para Datamoons Online deve seguir estas diretrizes.

---

# Filosofia Principal

Datamoons não são apenas monstros colecionáveis.

Datamoons são formas de vida híbridas, biológicas e digitais, originadas dos sonhos do Ser da Lua.

Cada Datamoon deve parecer uma criatura que realmente pertence ao Mundo dos Datamoons.

O objetivo não é criar apenas criaturas visualmente interessantes.

O objetivo é criar criaturas com propósito.

---

# Fórmula Oficial de Criação

Todo Datamoon deve ser criado utilizando a seguinte estrutura:

Base Natural
+
Conceito Principal
+
Influência Cultural
+
Tipo Fundamental

=

Datamoon

---

# Base Natural

A Base Natural representa a principal inspiração biológica da criatura.

Exemplos:

- Morcego
- Coruja
- Lobo
- Cobra
- Aranha
- Tubarão
- Formiga
- Raposa
- Camaleão

A Base Natural deve ser facilmente reconhecível mesmo após estilizações.

---

# Conceito Principal

Representa a ideia central que diferencia a criatura.

Exemplos:

- Som
- Memória
- Velocidade
- Gravidade
- Reflexos
- Sonhos
- Navegação
- Tempo
- Magnetismo
- Ecos

Todo Datamoon deve possuir apenas um conceito principal dominante.

Conceitos secundários podem existir, mas não devem competir com o conceito principal.

---

# Influência Cultural

Representa a inspiração obtida pelo Ser da Lua através do conhecimento humano.

Exemplos:

- Vampiros
- Dragões
- Piratas
- Ninjas
- Astronomia
- Mitologia Grega
- Mitologia Egípcia
- Folclore Brasileiro
- Tecnologia
- Inteligência Artificial

A influência cultural adiciona identidade à criatura.

---

# Tipo Fundamental

Todo Datamoon pertence a apenas um tipo.

## Datacore

Representa:

- Ordem
- Estrutura
- Conhecimento
- Memória
- Estabilidade

---

## Patch

Representa:

- Evolução
- Adaptação
- Correção
- Integração
- Equilíbrio

---

## Glitch

Representa:

- Mudança
- Ruptura
- Liberdade
- Imprevisibilidade
- Caos

---

# Estrutura Evolutiva

Toda espécie deve possuir potencial para seguir a estrutura:

Code
↓
Nex
↓
Omega

Nem todas as evoluções precisam estar definidas imediatamente.

Mas toda espécie deve nascer com uma direção evolutiva planejada.

---

# Estágio Code

Representa:

- Potencial
- Inexperiência
- Curiosidade
- Crescimento

Características comuns:

- Corpo pequeno
- Aparência amigável
- Poucas habilidades
- Grande capacidade de evolução

---

# Estágio Nex

Representa:

- Desenvolvimento
- Determinação
- Identidade

Características comuns:

- Aparência mais madura
- Maior domínio de habilidades
- Personalidade mais definida

---

# Estágio Omega

Representa:

- Realização
- Maturidade
- Domínio

Características comuns:

- Forma mais poderosa da espécie
- Expressão máxima do conceito da criatura

Omega não significa necessariamente:

- Gigante
- Violento
- Agressivo

Omega representa a conclusão do conceito.

---

# Papel Ecológico

Todo Datamoon deve possuir uma função dentro do ecossistema.

Pergunta obrigatória:

"Por que esta criatura existe?"

Exemplos:

- Predador
- Explorador
- Polinizador
- Guardião
- Escavador
- Migrador
- Coletor
- Dispersor de sementes
- Regulador populacional

Nenhuma espécie deve existir apenas por estética.

---

# Papel de Gameplay

Todo Datamoon deve possuir uma função clara em combate.

Exemplos:

- Atacante
- Glass Cannon
- Tank
- Bruiser
- Controle
- Suporte
- Mobilidade
- Utilidade

O papel deve ser identificável imediatamente.

---

# Personalidade

Toda espécie deve possuir tendências comportamentais naturais.

Exemplos:

- Curioso
- Tímido
- Territorial
- Protetor
- Agressivo
- Inteligente
- Sociável
- Independente

O tipo influencia a personalidade, mas não a define completamente.

Datamoons Glitch não são necessariamente malignos.

---

# Link

Todo Datamoon deve possuir alguma relação temática com o sistema de Link.

Perguntas recomendadas:

- Que tipo de jogador combina com esta criatura?
- Como ela demonstra vínculo?
- Como ela reage à confiança?

---

# Evolução

Toda evolução deve respeitar:

Link
+
Influência
+
Requisitos da espécie

Nível sozinho nunca deve ser suficiente.

---

# Influências Evolutivas

Toda espécie deve possuir potenciais influências evolutivas.

Exemplos:

- Anomalias
- Artefatos
- Regiões
- Eventos
- Fragmentos especiais
- Fenômenos lunares

Essas influências ajudam a definir futuras evoluções.

---

# Checklist de Criação

Antes de aprovar um Datamoon, confirmar:

[ ] Possui Base Natural

[ ] Possui Conceito Principal

[ ] Possui Influência Cultural

[ ] Possui Tipo Fundamental

[ ] Possui Papel Ecológico

[ ] Possui Papel de Gameplay

[ ] Possui Personalidade

[ ] Possui Direção Evolutiva

[ ] Possui Influências Evolutivas

[ ] Reforça a identidade do universo

---

# Exemplo Oficial

Nocmoon

Base Natural:
- Morcego

Conceito Principal:
- Som

Influência Cultural:
- Vampiro

Tipo:
- Glitch

Resultado:

Uma criatura noturna especializada em ecolocalização, vibrações sonoras e drenagem de energia.

Nocmoon é o exemplo de referência para aplicação destas regras.

---

# Regra Final

Se um Datamoon não puder ser explicado através de:

Base Natural
+
Conceito Principal
+
Influência Cultural
+
Tipo Fundamental

então seu design deve ser revisado.
